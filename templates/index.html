Key Features

Data Extraction

Automated scraping and API integration for up-to-date CVE vulnerability data collection.

Data Cleaning

Processes raw CVE data to ensure accuracy, consistency, and removal of duplicates.

Database Management

Securely stores processed data in MongoDB Atlas for quick access and scalability.

Interactive Visualization

Visualizes vulnerability data with Plotly for real-time insights and dynamic exploration.

NLP Integration

Uses GROQ with LLaMA 3.1 8B to interpret CVE descriptions and generate natural language insights.

Similarity Search

Finds top 5 similar CVEs for a given cyberattack description using RAG and vector search via Pinecone.

Project Overview

The CVE NLP Intelligence project automates data collection, cleaning, and analysis of cybersecurity vulnerabilities. It integrates NLP models for semantic search and language inference, combined with Flask, MongoDB Atlas, and Plotly visualizations to deliver a seamless and interactive vulnerability analysis platform.

  • Scraping and API-based CVE data ingestion pipeline.
  • Data cleaning and consistency checks for high-quality insights.
  • MongoDB Atlas backend for structured data storage and retrieval.
  • Flask-powered dashboard with intuitive data visualizations.
  • Integration with GROQ + LLaMA 3.1 for intelligent query answering.
  • Semantic similarity detection using RAG and Pinecone vector embeddings.
